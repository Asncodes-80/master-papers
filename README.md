# برگه‌های مربوط به کارشناسی ارشد مهندسی نرم‌افزار

تمارین، مقالات و جورنال‌ها، تحقیق‌های انجام شده در این رشته در دانشگاه آزاد
اسلامی واحد تهران و شمال.

## محتوا

شامل دروس و واحد‌های اختصاص داده شده در این دانشگاه می‌باشد. همچنین برخی دروس
مانند الگوریتم یا پردازش موازی، شامل بخش‌هایی برای حل تمارین و یا ارائه به صورت
کد نویسی می‌باشد.

## مشارکت

می‌خواهید در بهتر شدن محتوا کمک کنید؟ می‌توانید [بیشتر](./misc/README.md) 
بخوانید‌.

البته، با کمال میل و سرعت این کار رو برای شما انجام میدم! 😊

برای هر یک از سطوح انزاوی مورد نظر، یک جدول زمان‌بندی ارائه خواهم داد که میزان عدم انطباق اطلاعات بین تراکنش ها در آن به صورت Isolation مشخص شده باشد:

1. Read uncommitted:
- تراکنش A مقدار X را به صورت نامتعهدانه (uncommitted) به روز می کند.
- تراکنش B مقدار X را به صورت خوانده شده (read) می خواند، حتی اگر تراکنش A هنوز مقدار X را نهایی (committed) نکرده باشد.

|        | تراکنش A | تراکنش B |
|--------|-----------|-----------|
| شروع   |           |           |
| خواندن X توسط تراکنش B |         | X         |
| به روز رسانی X توسط تراکنش A | X       |         |
| پایان  | X         | X         |


2. Read committed:
- تراکنش A مقدار X را به صورت commit میکند
- تراکنش B مقدار X را می خواند، پس از آنکه تراکنش A X را Commit کرده است.

|        | تراکنش A | تراکنش B |
|--------|-----------|-----------|
| شروع   |           |           |
| به روزرسانی X توسط تراکنش A | X         |         |
| خواندن X توسط تراکنش B |         | X         |
| پایان  | X         | X         |


3. Repeatable read:
- تراکنش A مقدار X را می خواند و تراکنش B همان مقدار X را به صورت read می خواند ، حتی اگر تراکنش A در طول تراکنش B X را به صورت Commit کرده باشد.

|        | تراکنش A | تراکنش B |
|--------|-----------|-----------|
| شروع   |           |           |
| خواندن X توسط تراکنش A | X         |           |
| خواندن X توسط تراکنش B |           | X         |
| به روز رسانی X توسط تراکنش A | X       |           |
| پایان  | X         | X         |
